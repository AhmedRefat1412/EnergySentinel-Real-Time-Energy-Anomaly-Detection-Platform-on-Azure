## рџ“Њ Project Overview

Industrial machines generate continuous sensor readings (power, voltage, temperature, frequency, etc.). This platform simulates that telemetry, streams it through Azure, and:

- Stores raw data reliably (Bronze)
- Cleans and validates it (Silver)
- Aggregates it for reporting (Gold)
- Runs a trained ML model in real-time to flag anomalies before they escalate (Platinum)
- Surfaces insights through Power BI dashboards

## рџ”© Components

### 1. Producer (`/producer`)
Simulates 10 sensors across 3 plants, generating a reading every second (power, voltage, current, temperature, power factor, frequency) and injecting realistic anomalies (power spikes, voltage drops, overheating, frequency faults) at a ~5% rate. Sends events to Azure Event Hubs.

рџ“„ [How it was set up в†’](docs/steps/01_event_hub_and_producer.md)

### 2. Streaming Notebook (`/notebooks/streaming`)
Reads live events from Event Hubs (via Kafka-compatible protocol), writes raw data to **Bronze**, and simultaneously runs a trained LightGBM model on each micro-batch to detect anomalies in real-time, writing results to **Platinum**.

рџ“„ [How it was set up в†’](docs/steps/02_streaming_notebook.md)

### 3. Silver Notebook (`/notebooks/silver_notebook.py`)
Reads from Bronze, runs Data Quality checks (nulls, out-of-range values, unknown sensors, duplicates), and separates clean data (в†’ Silver) from invalid data (в†’ Quarantine).

рџ“„ [How it was set up в†’](docs/steps/03_silver_notebook.md)

### 4. Gold Notebook (`/notebooks/gold_notebook.py`)
Builds three aggregated tables: `plant_summary`, `sensor_summary`, and `hourly_trends`, ready to be queried by Synapse and visualized in Power BI.

рџ“„ [How it was set up в†’](docs/steps/04_gold_notebook.md)

### 5. ML Model (`/ml`)
LightGBM Classifier trained on 33,850 labeled readings (95% normal / 5% anomaly).

### 6. Orchestration вЂ” Azure Data Factory
Schedules the Silver and Gold notebooks to run automatically on a defined schedule, and runs periodic `OPTIMIZE`/`VACUUM` jobs on Bronze to compact the small files generated by streaming.

рџ“„ [How it was set up в†’](docs/steps/06_data_factory.md)

### 7. Serving Layer вЂ” Synapse Serverless SQL + Power BI
Synapse Serverless SQL exposes the Gold Delta tables via external tables/views, which Power BI connects to for a live, auto-refreshing dashboard.
